# MudBlazor 9.7.0 → 9.8.0 — upgrade rehearsal

**Status:** `v9.8.0` is not tagged upstream yet (latest: `v9.7.0`). Use this doc as the release ritual template.

## When Mud 9.8 ships

```bash
cd external/TheNerdCollective.Components/src/TheNerdCollective.MudComponents.DesignTokens
chmod +x scripts/diff-mudblazor-upgrade.sh
./scripts/diff-mudblazor-upgrade.sh 9.7.0 9.8.0
```

This writes `reference/mudblazor/upgrades/9.7.0-to-9.8.0.md` with added/removed `--mud-palette-*` keys from `MudThemeProvider`.

1. Update `MudBlazorPaletteManifest.cs` + `PALETTE-MANIFEST.md`
2. Map new slots in `NerdMudBlazorPaletteBindings` / token pack `frameworkDefaults.mudblazor.palette`
3. Run `scripts/harvest-mudblazor-inventory.sh`
4. Run unit tests + Playwright (`mud-palette-fidelity`, `mud-visual-regression`)
5. Refresh visual baselines under `reference/mudblazor/9.8.0/visual/`
6. Bump all `PackageReference Include="MudBlazor"` pins + `NerdDesignTokenOptions.MudBlazorVersion`
